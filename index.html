简介和思考人生

动机和第一个下午

许久之前就想做个个人博客，当时看的是Wordpress，安装包都下了但是因为各种拖延症迟迟没有动手，上次看到某个老乡使用Github Pages+Hexo搭建个人博客，用Markdown写。之前一直用Markdown，于是马上把这个信息记录到笔记里，今天（2016.5.20）整理笔记又翻到了，机不可失时不再来，所以玩了一下午把基础配置弄好了，传了两篇文章上去，很开心

在此开坑

为了防止大家说楼主没有下面，这篇文章会持续更新，记录我所有建站的思路、步骤、资料来源、感悟，所有的图片源于各种官网和我的截图

为什么你要写博客？

偶然看到的一篇文章，太有道理啦。

现学现卖介绍几个概念

  • Github 分布式版本控制系统\代码托管平台\开发者社区
  • Github Pages 基于Github免费构建一个静态网页
  • Hexo 一款基于Node.js的静态博客框架，支持Markdown
  • NexT 一种Hexo的主题，集成了一些第三方应用
  • Markdown 一种美妙的语言
  • 整个系统可以理解为：Git做免费服务器，Git还给你提供一个叫XXX.Github.io的免费域名，当然也可以用自己买的域名，用Hexo对整个博客进行设置和编辑，其中应用了NexT主题，全部内容用Markdown书写

本文与其他教程的主要区别

  1. 用便携版Hexo 网上各种教程里，在Windows下使用Hexo都要先安装NODEJS，NODEJS里面集成了NPM，用NPM来安装Hexo，然后通过命令行进行各种复杂的操作最后才能把网页搭建好（当然也有很多大神开发了很多高科技自动化部署代码，我并没有了解太多）。我安装Hexo的时候居然提示我的机器与它不兼容，搜索了一下也有人有类似的情况，但是无解。找到Hexo的QQ群(287306637)提问，结果热心的前辈们热情安利（Yiboguanggao）：使用便携版Hexo。直接一个绿色压缩包，集成了Git、NODEJS、NPM，另外装着几个批处理文件，稍微设置一下就成功了，对于我这种小白来说比网上的普通小白教程方便多了，在下面慢慢说
  2. 比他们Low很多 作为一个编程小白，我目前只会使用很接地气的语言来描述一些事情，而且我的方法可能很没有效率，很多情况下更偏向于做实验去验证而不是研究其原理，不过，我会用心去写的，并且正在努力变得更强XD

来动手吧！

开始Github之旅

创建Github账号

Github官网

创建Github Pages

  • 查看页面右上角的＋符号，New repository
  • 如果要做个人博客，repository名必须设置和Owner一样，即Owner.github.io，这是因为Github Pages有两类
    • User Pages：repository名与Owner相同，该repository中的内容会展示在Owner.github.io上
    • Repository Pages：repository名与Owner不同，假设该名为test，该repository中的内容会展示在Owner.github.io/test上
  • 勾选Initialize this repository with a README，点击Create repository
  • 进入该项目的Settings
  • 在Github Pages一栏点击Launch automatic page generator选择版式和主题，然后点Publish发布

  • 至此你就可以访问你的User Pages了

    关于图片的调用链接（图床）

  • 我原先是习惯在为知笔记里编辑Markdown笔记，图片则是本地导入或是直接用QQ截图Ctrl+Alt+A放进去的，现在想要发布到博客，问题来了，为知笔记里的图片URL是本地目录，文章上传到Github就无法调用了，而且为知笔记不能另存为成.md格式，有点麻烦
  • 最笨的方法是在Hexo\blog\source里建立一个image文件夹，把图片放到里面，在Markdown文章里一个个写这个图片的路径\image\xx.jpg，然后一并部署到Github，这太繁琐了，而且不方便博客的迁移
  • 如果能上传图片到云端，然后生成公开的URL随意调用就方便多了，于是我查了一下生成外链的工具，找到了国内广泛使用的七牛云存储，这里主要给大家介绍这个工具，七牛自带的同步工具并不好用，我使用极简图床
  • 后来我改用了MarkdownPad2编辑器，在它的官网下的，似乎可以直接给图片产生外链，不过这是付费功能
  • 现在我又下了个MarkdownPad2破解版。。说是可以使用完全的功能，还没试过。。试成功了我马上支持正版。。

    将图片上传至七牛云产生外链

    1. 在七牛云注册账号之后，进行实名认证，再绑定支付宝，可以获得10G永久免费空间

    2. 登陆控制台，在个人面板中查看密钥并记录，有AK和SK两个密钥

    3. 创建一个公开的资源空间，点击左侧的添加资源即可

    4. 将图片上传至七牛云

    方法1. 使用七牛客户端进行同步操作
    七牛开发者中心-qrsbox 图形界面上传同步工具
    直接打开qrsbox，进行同步配置，同步源目录可以设定为F:\hexo\qiniu。qrxbox启动后会在 Windows 的任务栏中显示托盘，它会自动将源目录与你创建的公开资源空间进行文件同步，可以登陆网页端查看它们，要在文章中插入图片时就在七牛云端复制外链做成Markdown语句即可，麻烦的是要一个个点击复制，

    如果上传失败
    参考HTTP状态码查找错误，我第一次上传失败反馈错误401，很容易就确定是密钥错误，查看一下配置文件是因为粘贴AccessKey的时候开头多了一个空格
    关于七牛客户端同步传输的实验
    在七牛客户端托盘运行的情况下，在源文件夹hexo\qiniu中做以下操作：
    • 添加新的文件XXX.XXX
    • 云端立即同步，命名为XXX.XXX
    • 在文件夹中的XXX.XXX重命名为YYY.YYY
    • 云端立即同步，原来的文件更名为YYY.YYY
    • 建立子文件夹hexo\qiniu\TEST，将hexo\qiniu中的图片YYY.YYY复制到该文件夹中
    • 云端立即同步，并没有生成子文件夹，而是将新的图片命名为TEST\YYY.YYY
    • 删除hexo\qiniu\YYY.YYY
    • 云端的文件YYY.YYY仍然存在
    • 在云端删除YYY.YYY，并将回收站中的YYY.YYY还原
    • 云端并未出现YYY.YYY
      方法2. 使用极简图床上传图片
      极简图床是一个基于七牛的第三方网页端上传工具，将图片拖入浏览器即可上传（可以一次拖进多个图片批量上传），在右上角进行七牛云密钥、域名的相关设置后可以自动同步至七牛云，然后自动生成Markdown图片语句，上传和引用都比直接用七牛客户端和云端方便多了

      目前我是先用QQ截图把图截下来直接粘贴到极简图床里，再把自动生成的Markdown语句粘到编辑器里，这个工具真是太棒了

问题与解决方案

  1. 用NPM装Hexo不成功，显示机器不兼容
    • 直接用便携版Hexo
  2. 上传到网页的中文文字变成乱码
    • 最好先把Hexo语言设置成中文。主要是因为编码不对，最好用sublime、notepad++等编辑器设置采用UTF-8编码，Windows最原始的notepad是要在保存\另存为的时候才可以设置UTF-8编码
  3. 部署成功了，打开网页还是上个版本
    • 等一会再打开，Github Pages有缓存

WIN7+UBUNTU14.04双系统（WIN7引导）安装教程

发表于 2016-05-20   |  

Title:WIN7+UBUNTU14.04双系统（WIN7引导）安装教程
Author:YAESUJ

准备工作

整个教程没有图你敢信，然而应该是好用的。
LINUX是一颗大树，有很多枝干，UBUNTU是其中一条。
这是我的百度云，先在这里把要用到的软件下好：
http://yun.baidu.com/pcloud/album/info?uk=1294310484&album_id=5057240083867339101
要下的是：

  • ubuntukylin-14.04.2（系统镜像文件）
  • Diskgenius（分区软件，x86表示32位系统，x64表示64位系统）
  • ULTRAISO （做系统盘的）
  • EASYBCD（设置引导菜单的，用于多系统启动的选择）

然后跟我一步步做：

阶段1，清理系统，分出空闲磁盘，制作系统盘

  • 进入WIN7系统
  • 分区
    • 打开Diskgenius（x86表示32位系统，x64表示64位系统），把CDE以外的都删除（清理原来装的其他系统，把盘符精简一下），选择E盘右键调整分区大小，余下约150G空闲空间用来装UBUNTU，最后要 点击左上角的保存设置，关闭
  • 【用ULTRAISO制作系统盘】准备一个空U盘，文件-打开-找到系统镜像文件（后缀名为ISO），启动-写入硬盘映像，格式化，点击write写入，这就将系统写入你的U盘了
  • 写入完毕后它还是能作为正常U盘使用的，你可以在U盘根目录上自己再建一个独立的文件夹把其他要用的文件装在里面

阶段2，用U盘装系统

  1. 开机的时候把装了系统的U盘插进去，不停摁F2进入BIOS界面（不同主板不一样，华硕主板一般是F2，可以自己百度）
  2. 选择boot菜单-把最下面的BOOT OPTION选项改成你的U盘（原来默认从硬盘启动，现在改为从U盘启动）
  3. ESC退出，保存设置（save changes）
  4. 重启电脑后进入UBUNTU启动界面（如果没进就重复2.），选择install ubuntu kylin进入UBUNTU安装界面
  • 到这里就要注意不要随便乱点下一步，看清楚再点，否则可能造成地球毁灭，最好断网安装
  • 选择语言 拉到最下面选择中文简体
  • 安装类型 界面选择其他选项（自己创建，调整分区）
  • 开始分区 点击阶段1分出来的空闲分区150G，双击编辑分区， 设置为：主分区，空间起始位置，用于：ext4文件系统，挂载点：/
    • LINUX新手就分这一个区就够了，多了也没什么用
  • 这里是关键，注意，可能造成太阳系毁灭 看清楚刚才产生的那个分区（挂载点：/）的标注，按照刚才的一套分区设置应该是SDA3（不过也不一定，看具体情况）
  • 在下面的 安装启动引导器的设备 选择这个号码（我现在是SDA3，你要看具体情况）。这样操作是把UBUNTU的引导放在/里面而不是修改整个磁盘的引导顺序，这样重启之后就是用WIN7引导UBUNTU
  • 开始安装 我这里名字写xd，密码123，勾选自动登录
  • 修改引导菜单 重启进入WIN7系统，打开EASYBCD，编辑引导菜单-把除了WIN7以外的都删除，添加新条目-linux/bsd-GRUB2-驱动器：150G那个区。超时选项：等待用户选择。保存设置，关闭

阶段3， 重启计算机，选择UBUNTU系统，开始使用UBUNTU

前期试验走了很多弯路，一共装了UBUNTU14、KALI2.0、WIN7三个系统，同志们一定要好好引导
